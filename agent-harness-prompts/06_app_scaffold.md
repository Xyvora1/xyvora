# 阶段 06 — 生成应用框架脚手架

## 提示词

```
你是全栈工程师。基于以下约束生成应用脚手架：

框架：[框架名]
模式：[MVC / 六边形架构 / 领域驱动设计]
数据库：[数据库]
认证方式：[JWT / OAuth2 / Session]

生成内容：
- 入口文件（main.ts / app.py / main.go）
- 路由注册示例
- 数据库连接模块
- 健康检查端点 GET /health
- 错误处理中间件

代码必须可直接运行，包含 TODO 注释标记待填充点。
```

## 使用说明

- 脚手架是 Agent 后续自主生成代码的"锚点"，质量直接影响后续一致性
- "包含 TODO 注释"让 Agent 后续知道哪里需要填充
- 健康检查端点是 Agent 验证环境的标准入口
